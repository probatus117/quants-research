# 中文化迁移分阶段实施计划

**目标文件:** `docs/plans/zh-localization-plan.md`

## Summary

将项目基础语言迁移为**简体中文且只保留中文自然语言入口**。迁移必须分阶段实施，每阶段只改一组边界清晰的文件，并在阶段末运行对应验证，避免一次性大改导致路由、测试、Agent prompt 和运行时文案混在一起失控。

不改代码标识符、API schema、环境变量、ticker、枚举值、数据格式和持久化路径。

## Phase Plan

- **Phase 0: 建立迁移基线**
  - 新增 `docs/plans/zh-localization-plan.md`。
  - 扫描日文/英文说明性文本、路由入口、测试断言和 prompt 文案。
  - 记录允许保留的英文技术词清单，例如 `risk-on`、`neutral`、`XAI_API_KEY`、`GraphRAG`。

- **Phase 1: 迁移 Codex canonical 资产**
  - 先改 `.agents/skills/*/SKILL.md`、`.agents/rules/*.md`、`.agents/agents/*/agent.md`。
  - 将角色说明、输出格式、操作规则、Reviewer/DeepThink 说明统一为简体中文。
  - 不改路由样例，先确保 agent 定义本身可读、完整。

- **Phase 2: 迁移中文路由入口**
  - 修改 `.agents/skills/stock-skills/routing.yaml` 和 `.agents/agents/*/examples.yaml`。
  - 将 `intent`、`triggers`、`header`、few-shot 示例全部切换为中文。
  - 不保留日文入口，例如 `いい株ある？` 改为 `有什么好股票？`。
  - 阶段验证：`conda run -n stock-skills-2 python tests/e2e/run_e2e.py --dry-run`。

- **Phase 3: 迁移运行时 prompt 和用户可见文案**
  - 修改 `src/`、`tools/`、`scripts/` 中的 docstring、注释、warning/error、Grok prompt、context formatter 输出。
  - Grok 研究类 prompt 默认要求中文输出，不再按日股/美股切换日文或英文 prompt。
  - 保持所有 dict key、返回 schema、函数名和文件格式不变。

- **Phase 4: 迁移测试和 fixture 基线**
  - 更新 `tests/test_kik746_dry_run.py`、`tests/e2e/test_scenarios.yaml`、`tests/e2e/test_mocked.py`。
  - 将测试输入、mock 输出、断言文本改为中文。
  - 更新 Grok prompt 测试，从“Japanese/English prompt”改为“统一中文 prompt”。
  - 阶段验证：`conda run -n stock-skills-2 python -m pytest tests/e2e/test_mocked.py -q`。

- **Phase 5: 迁移文档和 Claude mirror**
  - 修改 `README.md`、`AGENTS.md`、`CLAUDE.md`、`docs/*.md`、`config/*.yaml` 中的说明性文本。
  - 将 `.agents` 的结果同步到 `.claude`，保留 Claude Code 专用路径和平台名。
  - 阶段验证：`diff -qr .agents .claude`，只允许平台差异和已知结构差异。

## Final Verification

- `conda run -n stock-skills-2 python tests/e2e/run_e2e.py --dry-run`
- `conda run -n stock-skills-2 python -m pytest tests/e2e/test_mocked.py -q`
- `conda run -n stock-skills-2 python -m pytest tests/ -q`
- `rg -n "[ぁ-んァ-ン]" .agents .claude docs src tools tests scripts config AGENTS.md README.md CLAUDE.md`
- `rg -n "(Japanese prompt|English prompt|日本語|英語|日文|英文)" .agents .claude docs src tools tests scripts config AGENTS.md README.md CLAUDE.md`

## Assumptions

- 中文统一为简体中文。
- 实装时按 Phase 0 到 Phase 5 顺序推进，每阶段完成验证后再进入下一阶段。
- 每阶段建议单独提交，提交信息形如 `chore: migrate agent docs to chinese`。
- 不迁移 `data/cache`、`.pyc`、真实个人数据、许可证正文和第三方生成物。
